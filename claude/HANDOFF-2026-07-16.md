## 1. What this is

A multi-tenant shipping SaaS (FedEx-first). One giant React SPA in **`src/App.jsx`** (~14,300 lines), Tailwind via CDN (`index.html`), Netlify Functions backend (`netlify/functions/*`), Supabase (Postgres) for the cloud store. No component library — everything is hand-rolled primitives (`Input`, `Select`, `Field`, `Badge`, `Panel`, `Empty`, `Stat2`, `NumBox`, …).

**Two shipping backends:**
- **Direct FedEx** — `netlify/functions/fedex.js` (rates, transit, address, track, locations, pickup, cancel) + `ship.js` (booking). Uses `FEDEX_API_KEY`/`FEDEX_SECRET_KEY`.
- **England** (`england.rocksolidinternet.com`, a.k.a. RockSolid/REMS) — reseller backend for customers like **L'AGENCE**. `ship.js` proxies to `ship-england.js` when `CARRIER_BACKEND=england` / the customer has England creds. Rates come from England's API; **England does NOT quote One Rate up front** (important — see §5).

## 2. Build / deploy / conventions

- **Build:** `set -o pipefail; npm run build` (Vite). Warns about a 1.4 MB chunk — expected. `vite.config.js` now splits `react`/`icons`(lucide)/`vendor`(qrcode) into long-cached chunks so a redeploy only re-downloads the app bundle.
- **Version tag:** `const BUILD_TAG="addr-vNNN"` near the top of `src/App.jsx` (line ~112). **Bump it every release** — it's how you verify a deploy went live (poll the bundle for the tag).
- **Deploy = push.** Netlify auto-builds on push. Flow used all session:
  ```
  npm run build            # verify it compiles
  # bump BUILD_TAG, build again
  git add -A && git commit -m "..."
  git push -u origin claude/inspiring-gauss-iov8fe
  git push origin claude/inspiring-gauss-iov8fe:main
  git push origin claude/inspiring-gauss-iov8fe:staging
  ```
  Local `main`/`staging` are stale — **push the branch ref directly** (`branch:main`), don't `git checkout main` and merge.
- **Verify live:** `curl -s https://shippingcloud.net/ | grep -oE 'assets/index-[A-Za-z0-9_-]+\.js'` then curl that JS and grep for `addr-vNNN`.
- **Backend (function) changes** deploy on the same push. `node -c netlify/functions/foo.js` to syntax-check before shipping.

## 3. Key systems & where they live (src/App.jsx)

- **Persistence:** `usePersist(key, initial)` (~5500). Namespaced per login (`u/<uid>/<key>`), except `GLOBAL_KEYS` (shared: clients, rateRules, users…). `PERSIST_BUS[nsKey]` is a Set of setters so a store update pushes into all mounted components. `lsGet/lsSet/lsRaw` wrap localStorage under `sc_` prefix.
- **Cloud sync:** `cloudCall(payload, timeoutMs=15000)` → `/.netlify/functions/db`. `cloudFlush()` batches writes (`putMany`, 30 s timeout). `cloudPollTick` pulls shared stores every 20 s. `cloudLoadAll()` does a full `getAll` (45 s timeout — admin payloads are huge) and now pushes changed non-queued keys through `PERSIST_BUS` so a **background** refresh updates the UI live.
- **Boot (`App` → `start()`, ~6407):** **cache-first.** A returning cloud user with cached data renders **instantly** (`setPhase("ready")`) and verifies/syncs in the background — no blocking loader. Offline banner (`bootMsg`) only shows when `navigator.onLine===false` and is dismissible.
- **Rate engine:** `rateSellFor(cost, label, ctx)` (~2576) — the single source of truth for sell pricing. Base freight priced by the service rule (% / Fixed / Flat / List−%), fees priced by accessorial rules, account markup as fallback. Returns `cost` when nothing is configured (no markup); returns `null` only when `cost==null` and there's no flat rule. `quoteRates()` (~311) is the LOCAL estimate table (used as a fallback for the order modal on direct-FedEx-not-connected).
- **`showMoney`** (App ~6860) = `isDemo || isAdmin || featureOn("seeCosts", user, flags)`. **It gates the DASHBOARD cost/margin analytics — NOT a customer's own shipping charge.** The Shipments-tab rate is intentionally ungated (a customer's own bill always shows); only the "cost → margin" line inside the breakdown is `isAdmin`-gated.
- **Admin permission levels:** `ADMIN_LEVELS` / `ADMIN_SECTIONS` / `adminSectionsFor(user)` (~93). Levels: Full / Manager / Support / Billing / Custom. "All Logins" is always accessible so nobody locks themselves out. `BUILT_IN_ADMINS` (spencer@freightwire.com, spencer@shippingcloud.net) are always full admins.
- **Feature flags:** `FEATURE_CATALOG` (~75), `featureOn(id, user, flagsForUser)` (~104). Admins always true.
- **AI assistant:** `AssistantChat` (~6234). FAB bottom-right; the corner **X minimizes to a slim edge tab** (session-scoped, `scAiMini`), never fully hides. `AI_NAME` per brand.

## 4. HARD RULES (learned the hard way this session — do not violate)

1. **NEVER show a guessed/estimated delivery date or transit.** Only FedEx-committed transit (`fxLive` + `q.fxDays`/`q.fxDate` from the transit API). No `addBizDays`/`estTransitDays`/`minDays–maxDays` fallbacks shown to anyone — not the Ship list, not the buyer's Shopify checkout, not the admin preview. If FedEx returns nothing, show "—".
2. **NEVER show raw carrier cost as the rate**, and **never estimate a price.** The Shipments rate shows the stored **sell** (what the customer was quoted/charged). If an older shipment has no sell, reconstruct it from the **stored carrier cost × that customer's markup rule** (deterministic, same math the quote used) and **persist** it — that's not an estimate. If there's no stored cost either, it stays "—" (recover via England lookup, see open items).
3. **Cost is admin-only** — only the "Your cost → margin" line (isAdmin-gated).
4. **Don't assume Spencer's timezone.** Don't say "dawn"/"morning" etc.
5. Commit + push **before** any temp/screenshot build that does `git checkout` (it eats uncommitted work).

## 5. Why "blank rate in Shipment history" happened (context for future bugs)

L'AGENCE ships via **England**, which **doesn't quote One Rate up front** → the quote object had `sell=null`/`cost=null`, so the booked shipment stored no price → "—". Fixed in v610: `ship-england.js` now returns `cost` (England's `totalShippingCost`) on booking; `buildRec` (Ship tab) and the order-modal rec both store that real cost and mark it up into sell when the quote had none. **New bookings are fine.** Old blank ones (booked pre-v610) have no price saved — they need a one-time **England cost-recovery pull** by booking number (offered to Spencer, not yet built — see open items).

Also v611: the Shipments rate was wrongly gated behind `showMoney`, hiding a customer's own charge when viewing as that customer (impersonation). Now ungated.
