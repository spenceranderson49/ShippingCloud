Part B · What's actually missing — ranked

  1. USPS absent

    The single biggest real gap. For domestic ecommerce light parcels, USPS is the volume carrier — Ground Advantage / First-Class. Without it you can't seriously serve the largest slice of the market, and it's what every competitor leads with. England's reseller quoting references USPS, so England-connected accounts may see it — but you have no direct USPS label path.
    Fix: add USPS via a consolidator/aggregator (EasyPost or Shippo) — one integration also finishes UPS & DHL, plus discounted rates. Fastest path by far.

  2. Finish UPS labels + add DHL rating only

    UPS is already wired for OAuth + live rating — but not label buying. DHL has no direct integration. You're one step from true own-account multi-carrier.
    Fix: either complete the UPS Shipping API label call (you're ~halfway), or let the aggregator layer cover all three at once.

  3. Rate-shopping across carriers in the UI

    Your engine and API can quote per-carrier, but the app doesn't yet "shop all carriers and auto-pick the cheapest that hits the delivery date." That's a headline selling point for ShipStation/Veeqo/Shippo. Unlocks the moment you're multi-carrier.
    Fix: a "best rate" column + Autopilot rule "cheapest by delivery date." Small once carriers exist.

  4. FedEx Track API entitlement ops, not code

    Your tracking sync + branded pages + webhooks are fully coded but dormant — they need the "Track API" product enabled on England's FedEx developer project. Until then, statuses don't flow.
    Fix: England enables the Track API product on the FedEx portal. Zero code.

  5. Returns: exchanges + package protection

    You have a branded returns portal + restock. The revenue features competitors monetize — instant exchanges, store credit, and package protection/shipping insurance upsell (Redo/Route) — aren't there yet.
    Fix: add exchange/store-credit flows to the returns portal, and a protection line item at checkout/label.

  6. Third-party shipment insurance

    You support declared value (carrier insurance). Cheaper third-party insurance (Shipsurance-style) is margin for you and standard in ShipStation/Shippo/EasyPost.
    Fix: rides on the EasyPost/Shippo layer — insurance is a flag on the label call.

  7. International DDP / landed-cost prepay

    HS codes + commercial invoices exist; prepaid duty & tax (Delivered Duty Paid) does not. Easyship's whole edge. Only matters if cross-border is a target.
    Fix: landed-cost calc at checkout; the aggregators expose DDP too.

  8. Customer marketing / SMS / review requests

    Post-purchase email exists; SMS notifications and review-request/marketing flows (ShippingEasy, ReadyCloud) don't. Retention & upsell surface.
    Fix: add SMS to the notification engine; a review-request trigger on delivery.

  9. Deeper carrier-performance analytics

    You have margin/shipment analytics. Carrier on-time SLA, cost-savings vs list, and delivery-performance dashboards would round it out.
    Fix: derive from the tracking data you already collect once the Track API is live.

Part C · My honest strategy — how you actually win

Don't out-breadth ShipStation. Own the combination nobody else nails.

ShipStation, Shippo and EasyPost win on carriers and channels but are thin on warehouse and non-existent on a 3PL/reseller back-office. ShipHero and Deposco win on WMS but are weak on multichannel shipping breadth and reseller billing. You are the only one building all three in one system: multi-channel shipping + a real WMS + a 3PL/agency back-office (the ELEMS replacement).

That's a "run your entire shipping business" story aimed squarely at 3PLs, shipping resellers, and agencies — England's exact model. Lean into that positioning; it's defensible and the incumbents can't easily copy it.

The build order I'd commit to:

  1. Add the EasyPost (or Shippo) carrier layer. One move delivers USPS + full UPS + DHL + rate-shopping + third-party insurance + more address/DDP tooling. This is the unlock — nothing else competes without it.
  2. Ship the "best rate" rate-shopping UI + Autopilot rule. Turns multi-carrier into a headline feature.
  3. Get England to enable the FedEx Track API product. Free; lights up tracking, branded pages, and webhooks you already built.
  4. Returns exchanges + package protection. Revenue and retention.
  5. Then international DDP and SMS/marketing — once the core is unbeatable.
