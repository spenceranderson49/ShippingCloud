Very specific things to build — ranked

  1. Add EasyPost (or Shippo) as a carrier layer do first · unlocks 6 gaps

    The single highest-leverage build. One integration gives you USPS + full UPS + DHL + rate-shopping + third-party insurance + international address validation + DDP hooks — closing six scorecard rows at once.
    Build: netlify/functions/easypost.js → POST /v2/shipments (returns rates from all carriers), /buy (purchase), /refund (void), insurance flag. · Settings → Carriers: an "EasyPost/Shippo API key" connect field. · Route the Ship-screen rate call through it alongside FedEx. Done when: you can buy a real USPS Ground Advantage label from the Ship tab.

  2. "Shop all carriers" rate panel + cheapest-by-date Autopilot rule 1 wk after #1

    Turn multi-carrier into the headline feature every competitor sells on.
    Build: On the Ship screen, a rate list showing every carrier/service (FedEx + EasyPost carriers) sorted by price with delivery date + Buy. · New Autopilot rule type: cheapest service arriving by {N} days. · Add a "best rate" column to Orders. Done when: a rule auto-picks the cheapest label that still hits the promised date.

  3. Turn on the FedEx Track API product ops · zero code

    Your tracking sync, branded pages and tracking.updated webhooks are fully built but dormant.
    Do: England enables the "Track API" product on the FedEx developer project. Lights up tracking + branded tracking + webhooks instantly. Done when: a shipment shows live scan events on the branded page.

  4. Returns → exchange-first + store credit revenue

    Copy Redo's mechanic: steer refunds into exchanges/store credit. Redo reports exchanges up ~78%.
    Build: Extend ReturnsPortal — an exchange flow (swap for another variant, auto-create the replacement order), a store-credit option, and configurable return reasons/rules. Done when: a customer can exchange a size without a refund.

  5. Shopper-funded package protection margin

    Redo's "claims-fund" model turns coverage into a revenue line, not a cost.
    Build: an optional protection fee (~$0.98-1.98, scaled to order value) at checkout/label that accrues to a merchant claims fund; a simple claims-approval queue in admin. Done when: protection revenue and a claim can both be recorded.

  6. SMS notifications + review requests retention

    ReadyCloud/ShippingEasy monetize post-purchase messaging; you only have email.
    Build: add a Twilio SMS channel to the notification engine; triggers on shipped/out-for-delivery/delivered; a review-request send N days after delivery. Done when: a delivery fires an SMS + a review ask.

  7. Landed-cost / DDP calculator international

    Easyship's flagship — duties/taxes shown/prepaid at checkout cut cart abandonment. You already have HS codes.
    Build: a landed-cost estimate (duty + tax by destination + HS code) on international quotes; a DDP toggle that prepays via the carrier layer. Done when: an international quote shows fully-landed cost.

  8. SmartRate-style predicted delivery dates polish

    EasyPost SmartRate shows transit-time percentiles, not just price — you get it free once on EasyPost.
    Build: surface predicted delivery date + confidence in the rate list; let the cheapest-by-date rule use it. Done when: rates show "arrives by" with a confidence.

  9. Optimized pick paths + 3D bins in the WMS WMS edge

    SkuSavvy's differentiator; ShipHero's AI picking claims 20-30% less walking.
    Build: order the pick list by bin-location path; a simple warehouse map view; batch by item similarity. Done when: a multi-order pick walks in location order.

  10. Dropship auto-split by supplier OMS

    Ordoro's standout — flag dropship SKUs, split multi-supplier orders, route POs to vendors.
    Build: a "dropship" flag on items with a supplier; on such orders, auto-split and generate a vendor PO per supplier. Done when: a mixed order splits into a warehouse shipment + a vendor PO.

How you actually win

Own the combination — then win on the things they're all bad at.

No competitor pairs multi-carrier shipping + a real WMS + a 3PL/reseller back-office (ELEMS) in one system. ShipStation/Shippo/EasyPost are thin on WMS and have no reseller billing. ShipHero/Deposco are weak on multichannel shipping breadth. That's your defensible wedge — the "run your entire shipping business" platform for 3PLs, resellers and agencies (England's model).

  • Win on billing transparency + support. Every single competitor — Shippo, EasyPost, Easyship, ShipStation, Veeqo, ShipHero — gets hammered on billing disputes and slow support. Make that your brand promise.
  • Poach ShipStation's developers. They moved API access behind higher tiers in 2025 (≈10× for API users). You have a full public API — market directly to that burned segment.
  • Then close carriers. Everything above is undermined until USPS/UPS/DHL ship. Do action #1 first.
